# 使用指南 — 从零到运行

> 每一步都有截图说明，跟着操作即可。

---

## 一、启动环境

### 步骤 1：启动 Docker Desktop

按 `Win` 键，输入 `Docker Desktop`，回车打开。

等待任务栏右下角出现小鲸鱼图标，且不再转圈（表示已就绪）。

![image](https://github.com/tuiz1/work.c/assets/placeholder)

> 如果鲸鱼一直是黄色/红色，说明 Docker 启动失败，重启电脑后再试。

### 步骤 2：打开 Git Bash

在 `D:\work_c` 文件夹空白处 **右键** → **Open Git Bash here**。

或者按 `Win+R`，输入 `bash`，然后 `cd /d/work_c`。

### 步骤 3：确认容器是否存在

```bash
docker ps -a | grep oslab
```

- **如果看到一行结果**：容器已创建，跳到步骤 5
- **如果没有结果**：容器还没创建，继续步骤 4

### 步骤 4：创建容器（仅第一次需要）

```bash
MSYS_NO_PATHCONV=1 docker run -d --name oslab -v "D:\work_c":/work -w /work ubuntu:22.04 sleep infinity
```

然后安装编译工具：

```bash
MSYS_NO_PATHCONV=1 docker exec oslab apt update
MSYS_NO_PATHCONV=1 docker exec oslab apt install -y build-essential gdb
```

> 这一步需要联网下载，约 2-5 分钟。

### 步骤 5：启动容器

```bash
MSYS_NO_PATHCONV=1 docker start oslab
```

如果容器已经在运行：

```bash
MSYS_NO_PATHCONV=1 docker exec oslab echo "容器正在运行"
```

## 二、编译代码

### 方法 A：一键编译（推荐）

```bash
MSYS_NO_PATHCONV=1 docker exec -w /work oslab make
```

输出：
```
gcc -Wall -Wextra -g -c calculate.c
gcc -Wall -Wextra -g -o client client.c calculate.o
gcc -Wall -Wextra -g -o server server.c calculate.o
gcc -Wall -Wextra -g -o server_process server_process.c calculate.o
gcc -Wall -Wextra -g -o server_thread server_thread.c calculate.o -lpthread
gcc -Wall -Wextra -g -o server_threadpool server_threadpool.c calculate.o -lpthread
gcc -Wall -Wextra -g -o server_threadpool_cv server_threadpool_cv.c calculate.o -lpthread
gcc -Wall -Wextra -g -o stress_test stress_test.c calculate.o -lpthread -lm
```

### 方法 B：使用快捷脚本

```bash
./oslab.sh make
```

### 方法 C：进入容器手动编译

```bash
MSYS_NO_PATHCONV=1 docker exec -it -w /work oslab bash
make
```

### 编译单个文件

```bash
MSYS_NO_PATHCONV=1 docker exec -w /work oslab gcc -o server server.c calculate.c -Wall
MSYS_NO_PATHCONV=1 docker exec -w /work oslab gcc -o client client.c calculate.c -Wall
```

## 三、运行服务器

### 启动服务端（任选一个）

打开 **第一个** Git Bash 窗口：

```bash
# 实验1：单进程服务器（一次只能处理一个客户端）
MSYS_NO_PATHCONV=1 docker exec -it -w /work oslab ./server

# 实验2：多进程服务器
MSYS_NO_PATHCONV=1 docker exec -it -w /work oslab ./server_process

# 实验2：多线程服务器
MSYS_NO_PATHCONV=1 docker exec -it -w /work oslab ./server_thread

# 实验3：线程池服务器
MSYS_NO_PATHCONV=1 docker exec -it -w /work oslab ./server_threadpool
```

看到以下输出表示启动成功：
```
服务器启动，监听端口 8888 ...
```

> 这个窗口会一直运行，**不要关**。
>
> 按 `Ctrl+C` 可以停止服务器。

### 为什么用 `-it` 而不是只用 `exec`？

`-it` 让你可以按 `Ctrl+C` 停止服务器。不加 `-it` 的话，服务器在后台跑，不方便停止。

## 五、调试代码

### 使用 GDB 调试

```bash
# 先编译（带 -g 参数，已默认开启）
MSYS_NO_PATHCONV=1 docker exec -w /work oslab make

# 启动 GDB
MSYS_NO_PATHCONV=1 docker exec -it -w /work oslab gdb ./server
```

常用 GDB 命令：

| 命令 | 作用 |
|------|------|
| `b main` | 在 main 函数设置断点 |
| `b calculate` | 在 calculate 函数设置断点 |
| `b server.c:92` | 在 server.c 第 92 行设置断点 |
| `r` | 运行程序 |
| `n` | 单步执行（不进入函数）|
| `s` | 单步执行（进入函数）|
| `c` | 继续执行到下一个断点 |
| `p buff` | 打印变量 buff 的值 |
| `p n` | 打印变量 n 的值 |
| `bt` | 查看调用栈（崩溃时很有用）|
| `q` | 退出 GDB |

### 调试示例

```bash
(gdb) b calculate              # 在计算函数设断点
(gdb) r                        # 运行服务器
# 此时另开终端启动 client，发送表达式...
# 服务器停在 calculate 入口
(gdb) p expr_str               # 查看收到的表达式
(gdb) n                        # 单步执行
(gdb) n
(gdb) p v                      # 查看计算结果
(gdb) c                        # 继续运行
```

## 八、容器管理

### 常用命令速查

```bash
# 查看所有容器状态
MSYS_NO_PATHCONV=1 docker ps -a

# 启动已停止的容器
MSYS_NO_PATHCONV=1 docker start oslab

# 停止容器
MSYS_NO_PATHCONV=1 docker stop oslab

# 重启容器（改了镜像配置后）
MSYS_NO_PATHCONV=1 docker restart oslab

# 进入容器shell（环境没坏时用这个排查）
MSYS_NO_PATHCONV=1 docker exec -it oslab bash

# 查看容器日志
MSYS_NO_PATHCONV=1 docker logs oslab
```

### 环境坏了怎么办

```bash
# 删除旧容器
MSYS_NO_PATHCONV=1 docker rm -f oslab

# 重新创建（代码不会丢，因为代码在宿主机 D:\work_c）
MSYS_NO_PATHCONV=1 docker run -d --name oslab -v "D:\work_c":/work -w /work ubuntu:22.04 sleep infinity
MSYS_NO_PATHCONV=1 docker exec oslab apt update
MSYS_NO_PATHCONV=1 docker exec oslab apt install -y build-essential gdb
```

## 九、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `docker: command not found` | 没开 Docker Desktop | 打开 Docker Desktop |
| `Error response from daemon` | Docker 服务异常 | 重启 Docker Desktop |
| `无法连接到服务器` | 服务器没启动 | 先启动服务端再开客户端 |
| `bind: Address already in use` | 上次服务器没正常退出 | 等 60 秒或加 SO_REUSEADDR（已默认添加） |
| `exec: "bash": executable file not found` | 命令格式问题 | 确保加了 `MSYS_NO_PATHCONV=1` 前缀 |
| `连接被拒绝` | 服务器没运行或端口不对 | 确认服务器 `./server` 在执行 |
